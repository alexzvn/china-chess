## Problem Statement

There is no online multiplayer Chinese Chess (象棋/中国象棋) game application. Users want to play Chinese Chess against other real people over the internet, with a polished UI, real-time gameplay, and proper rule enforcement — all accessible from a web browser.

## Solution

Build a full-stack Chinese Chess game with:
- A **Bun + ElysiaJS** backend server providing WebSocket-based real-time multiplayer, room management, and authoritative game rule validation.
- A **Vue 3 + Vite** frontend compiled to static assets served by the Bun server, styled with **Tailwind CSS** and enhanced with **VueUse/core** composables.
- A **pure TypeScript Chinese Chess engine** that validates all piece movements, check, checkmate, and stalemate detection.

Players join from a web browser, browse a lobby of open rooms, join or create a room, and play Chinese Chess in real time against each other with visual feedback (legal move hints, check indicators, turn highlighting), a chat channel, and sound effects.

## Implementation Decisions

### Architecture

- **Single-package monorepo**: One `package.json`, one `bun.lock`, one `node_modules/`. Vue source in `src/vue/`, server source in `src/server/`.
- **Bun as sole package manager**: `bun install`, `bun run`, `bun test`.
- **ElysiaJS** for HTTP routing, static file serving, and WebSocket management.
- **Vue 3 + Vite** for the frontend, using Composition API with `<script setup>`, plain `ref`/`computed` (no Pinia), and Vue Router in history mode.
- **Vite builds to `public/`**, which Elysia serves as static files. WebSocket upgrades happen on the same port.

### Game Engine Module

- Pure TypeScript module, framework-agnostic, no Elysia or Vue imports.
- Board representation: 2D array `board[rank][file]` (10 ranks × 9 files), each cell is `null` or a Unicode piece character.
- Coordinates: 0-indexed `{ rank: number, file: number }`.
- Per-piece validation functions: `isValidMove(pieceType, from, to, board)` for each of the 7 piece types (車, 馬, 炮, 士, 象, 將/帥, 兵/卒).
- Check detection: simulate move, scan entire board for enemy attacks on the king.
- Checkmate/stalemate detection: iterate all pieces of the current turn's color, check if any has legal moves.
- Move history tracking: full `Move[]` array recorded for each game.

### WebSocket Protocol Module

- JSON-based, action-oriented messages.
- Client → Server: `move`, `resign`, `drawOffer`, `drawAccept`, `drawDecline`, `chat`.
- Server → Client: `boardUpdate`, `gameStart`, `gameEnd`, `error`, `roomJoined`, `lobbyUpdate`.
- Anonymous connections: server generates `clientId` (nanoid 7 chars) on WebSocket open.
- First client message determines routing: `joinLobby` or `joinRoom`.
- Room ID: nanoid(7), URL-safe, stored in URL path (`/room/:id`).

### Room Management Module

- Room state: `{ roomId, createdAt, playerA, playerB, status }` where status is `waiting` | `playing` | `finished`.
- Lobby: list of rooms with `status === 'waiting'`.
- Room creation: first player becomes `playerA`, room enters `waiting` state.
- Room joining: second player becomes `playerB`, room remains `waiting` until both confirm start.
- Game start: both players click "Start Game", server randomly assigns Red/Black, room enters `playing` state.
- Grace period: 60 seconds for reconnection after disconnect; forfeit if timeout expires.
- In-memory state only; no persistence.

### Vue Frontend Module

- Components: `Board.vue`, `Piece.vue`, `RoomCard.vue`, `ChatPanel.vue`, `PlayerInfo.vue`, `Lobby.vue`, `Game.vue`.
- Tailwind CSS for all styling — utility-first approach, no custom CSS files.
- VueUse/core for reusable composables (`useLocalStorage`, `useMediaQuery`, `useWindowSize`, `useMouse`, `useElementSize`, `useThrottleFn`, `useFullscreen`, etc.).
- Responsive board that scales to fit any viewport (Tailwind responsive utilities + CSS `vmin`/`vw` units).
- Adaptive layout: chat alongside board on desktop, below on mobile (Tailwind `md:` breakpoint).
- Visual feedback: selected piece highlight, legal move dots (small circles for empty, rings for capture), red glow on king for check, active player area highlight.
- Sound effects via Web Audio API: piece placement click, capture thud, check warning tone.
- Game over: status bar below board with result message and "Back to Lobby" button.
- Auto-reconnect: WebSocket reconnects every 3 seconds with "Reconnecting..." status.

### Server Module

- ElysiaJS app with static file serving (`public/`) and WebSocket endpoint (`/ws`).
- CORS enabled only in development mode.
- Ping/pong heartbeat via Elysia's built-in WebSocket support (30s ping interval, 60s pong timeout).
- Structured JSON logging for room lifecycle, player actions, and chat messages.
- Error handling: WebSocket messages for game errors, HTTP status codes for infrastructure errors.
- No rate limiting for v1.
- Minimal env vars: `PORT`, `NODE_ENV`.

## Testing Decisions

### What makes a good test

- Test **external behavior**, not implementation details. For the engine, test that given a board state and a move, the result is correct (move applied or rejected, check detected, game over detected).
- For the WebSocket server, test that given client messages, the server produces the correct broadcast messages and state transitions.
- For the Vue frontend, test that given user interactions (click, type), the UI renders the correct state.
- Tests should be **independently verifiable** — each test should assert one specific behavior.
- Use `bun test` for server and engine tests (native TypeScript support, fast).
- Use Vitest for Vue component and E2E tests.

### Modules to be tested

1. **Chinese Chess Engine** — All piece movement rules, check detection, checkmate detection, stalemate detection, move validation (no self-check), legal move enumeration.
2. **WebSocket Server** — Room creation, room joining, game state machine transitions (waiting → playing → finished), move broadcasting, disconnect handling, reconnection grace period, resign/draw workflows.
3. **Vue Frontend** — Lobby rendering, room navigation, board rendering, piece selection, legal move highlighting, chat message display, game over display, responsive layout, reconnection UI.

### Test priorities

- **High priority**: Engine unit tests — the game rules are the most complex and domain-specific part. Get these right first.
- **Medium priority**: WebSocket integration tests — verify the server correctly routes messages and manages room state.
- **Lower priority**: Vue E2E tests — verify the full user flow works end-to-end.

## Out of Scope

- **AI opponent / single-player mode** — multiplayer only for v1.
- **Player names / accounts / authentication** — anonymous play with color identification only.
- **Room password protection** — rooms are joinable by anyone who knows the room ID.
- **Rating system / Elo** — no player ranking for v1.
- **Game replay / move history replay** — move history is tracked but not replayable.
- **Tournament mode / multiple simultaneous games per player** — one game per connection.
- **Mobile app (native)** — web browser only.
- **Database / persistence** — all state is in-memory.
- **Redis / horizontal scaling** — single process, single port.
- **Time controls / clock** — no chess clock for v1.
- **Sound settings / volume controls** — sounds are always on, no settings page.
- **Profanity filtering / chat moderation** — no message filtering for v1.
- **Draw by three-fold repetition / perpetual check rules** — only agreement-based draws for v1.
- **Undo / takeback moves** — moves are final once sent.
- **Production deployment infrastructure** — Docker, CI/CD, load balancing are out of scope.

## Further Notes

### Chinese Chess Piece Set

Red pieces (moves first): 車 馬 炮 士 象 帥 兵
Black pieces: 車 馬 砲 士 象 將 卒

Note: 炮/砲 are both used for the cannon piece. Red uses 炮, Black uses 砲 (common convention).

### Board Layout

- 9 files (columns), 10 ranks (rows).
- River (楚河漢界) between rank 4 and rank 5.
- Palace: 3×3 area with diagonal lines on each side (ranks 0-2 for Red, ranks 7-9 for Black).
- Starting positions: standard Chinese Chess setup.
- Point dots at traditional positions (8 per half).
