## What to build

A player can create a room from the Vue lobby. The server generates a `roomId` (nanoid 7), stores the room in-memory, and broadcasts the updated room list to all lobby-connected clients. The `Lobby.vue` component shows a list of open rooms (each with room ID and player count) and a "Create Room" button. Clicking Create Room sends `{ action: "createRoom" }` over WebSocket; server responds with `{ type: "roomCreated", roomId }` and broadcasts `{ type: "lobbyUpdate", rooms }` to all lobby clients.

## Acceptance criteria

- [ ] In-memory room store on server: `Map<string, Room>` where `Room = { roomId, createdAt, playerA, playerB, status }`
- [ ] WebSocket message `{ action: "createRoom" }` creates a room with `playerA` set to sender's `clientId`, `status: "waiting"`
- [ ] Server responds with `{ type: "roomCreated", roomId }` to the creator
- [ ] Server broadcasts `{ type: "lobbyUpdate", rooms: Room[] }` to all clients currently in the lobby
- [ ] `Lobby.vue` renders a list showing each room's `roomId` and `"1 player"` status
- [ ] "Create Room" button is visible and functional
- [ ] Opening two tabs: creating a room in tab 1 makes it appear in tab 2's lobby list
- [ ] Full lobby flow integration test using `bun test` (server-only, no browser)
