## What to build

Extend the game engine from #05 with check detection, legal move filtering (a move is illegal if it leaves your own king in check), checkmate detection (king in check + no legal moves = loss), and stalemate detection (no legal moves but king not in check = loss). Exports `isInCheck(board, color)`, `getLegalMoves(board, pos)`, `isCheckmate(board, color)`, `isStalemate(board, color)`, `makeMove(state, from, to)` which returns the new game state. The `makeMove` function validates the move, applies it, switches turns, and returns the new state. Full unit test coverage.

## Acceptance criteria

- [ ] `isInCheck(board, 'red')` returns `true` when Red's king is under attack by any Black piece
- [ ] `getLegalMoves(board, pos)` returns only moves that do not leave the moving player's king in check
- [ ] `isCheckmate(board, 'red')` returns `true` when Red is in check and has zero legal moves
- [ ] `isStalemate(board, 'black')` returns `true` when Black has zero legal moves but is not in check
- [ ] `makeMove(state, from, to)` validates the move, applies it, switches `turn`, increments move count
- [ ] Unit test: generic check (opponent 車 attacks king, any move that doesn't block or capture the 車 is excluded from legal moves)
- [ ] Unit test: checkmate position recognition
- [ ] Unit test: stalemate position recognition
- [ ] Unit test: makeMove returns the correct resulting board state
