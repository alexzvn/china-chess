## What to build

Full end-to-end move execution: player selects a piece, clicks a legal destination, the Vue client sends `{ type: "move", from, to }` over WebSocket to the server. The server validates the move using the game engine (piece rules + no self-check), updates the game state, broadcasts `{ type: "boardUpdate", board, turn, lastMove }` to both players, and appends the move to the move history. Both players see the updated board. The server enforces turn order — only the current player may send moves.

## Acceptance criteria

- [ ] Client sends `{ type: "move", from: {rank, file}, to: {rank, file} }` on legal square click
- [ ] Server validates: is the client the current turn's player? Is the move in legal moves?
- [ ] Invalid move → server responds with `{ type: "error", code: "INVALID_MOVE" | "NOT_YOUR_TURN", message }`
- [ ] Valid move → server updates state, switches `turn`, broadcasts `{ type: "boardUpdate", board, turn, lastMove, moveCount }`
- [ ] Move is appended to move history array on the server
- [ ] Both players see the board update in real time (two tabs show synchronized board)
- [ ] After a move, the turn switches and the other player's pieces become selectable
- [ ] Integration test: connect two clients, both join and confirm start, send a move from Red, verify broadcast reaches both
