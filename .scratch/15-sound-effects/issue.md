## What to build

A Vue composable `useSound()` that generates short sound effects programmatically using the Web Audio API — no external audio files. Three sound types: `move` (short click/tap when a piece is placed), `capture` (deeper thud when a piece is captured), `check` (higher-pitched warning tone when king enters check). Sounds are triggered by the Vue client when receiving `boardUpdate` from the server: if a piece was captured → play `capture`; if the new position is check → play `check`; otherwise → play `move`. Sounds play immediately via `AudioContext`.

## Acceptance criteria

- [ ] `useSound()` composable creates and manages an `AudioContext`
- [ ] `playSound('move')` produces a short, subtle click/tap sound (e.g., filtered noise burst, <100ms)
- [ ] `playSound('capture')` produces a deeper, slightly longer thud (e.g., low-frequency oscillator + noise)
- [ ] `playSound('check')` produces a higher-pitched warning tone (e.g., two-note ascending beep)
- [ ] Sounds are triggered reactively in `Game.vue` when processing `boardUpdate`: capture → capture sound, check → check sound, normal move → move sound
- [ ] `AudioContext` is resumed on first user interaction (browser autoplay policy compliance)
- [ ] No external audio files — all sounds generated via Web Audio API oscillators, gains, and noise
- [ ] Sounds play at a comfortable volume — not startling
