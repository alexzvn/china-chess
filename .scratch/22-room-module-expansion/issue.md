## What to build

Add two new functions to `src/server/rooms.ts` and update return types on existing functions. This keeps the Room module as the sole owner of room state mutations.

**New functions:**
- `resign(roomId: string, clientId: string): Room` — sets `room.status = "finished"`, determines winner by color, returns the room
- `leaveRoom(roomId: string, clientId: string): Room` — if host leaves, deletes the room from the map. If playerB leaves, resets room to `waiting` (clears gameState, colors, rematch flags, ready states)

**Return type changes:**
- `kickPlayer` returns `{ room: Room; kickedId: string }` instead of `Room` (caller needs the kicked player's ID for notifications)
- `toggleReady` returns `{ room: Room }` instead of `{ room: Room; gameStarted: boolean }` (adapter checks `room.status` instead)

## Acceptance criteria

- [ ] `resign(roomId, clientId)` correctly sets game status and determines winner
- [ ] `leaveRoom(roomId, clientId)` handles both host-leave (delete room) and playerB-leave (reset room)

- [ ] `toggleReady` returns `{ room }` — caller determines game start by checking `room.status`
