## What to build

Create `src/server/actions/phase.ts` with 2 action adapters. These handle post-game transitions and reconnection.

**Adapters:**
- `handleRematch(ctx: RoomActionContext)` — calls `rematch(roomId, clientId)`, if both players accepted, sends `roomUpdate` to both (reset to waiting), resets rematch flags
- `handleRejoinRoom(ctx: RoomActionContext)` — sends `gameStart` + `boardUpdate` to reconnecting player, sends `opponentReconnected` to opponent. Returns error if no active game in room.

## Acceptance criteria

- [ ] Both adapters implemented in `src/server/actions/phase.ts`
- [ ] `handleRematch` handles single-accept (sends `rematchState`) and double-accept (resets room to waiting)
- [ ] `handleRejoinRoom` sends correct game state to reconnecting player
- [ ] `handleRejoinRoom` returns error if room has no active game
- [ ] Unit tests for each adapter
